# نظام إدارة بنوك الدم

تطبيق سطح مكتبي لإدارة بنوك الدم يشمل: المخزون اليومي، التقارير الشهرية، كشوف العاملين، شيت الجاهزية، وإدارة الأجهزة.

## التقنيات المستخدمة

- **الخلفية**: Node.js, Express, lowdb (JSON), xlsx
- **الواجهة**: Vanilla JS, CSS, Font Awesome
- **التشفير**: bcryptjs, JWT
- **الأمان**: helmet, express-rate-limit, DOMPurify

## هيكل المشروع

```
blood-bank-server/
├── server.js              # السيرفر الرئيسي (Express)
├── jsondb.js              # قاعدة البيانات (lowdb wrapper)
├── init-db.js             # تهيئة قاعدة البيانات
├── package.json
├── data/
│   ├── db.json            # جميع البيانات
│   └── جاهزية بنوك الدم.xlsx
├── public/
│   ├── index.html         # الصفحة الرئيسية
│   ├── css/style.css      # التنسيقات
│   └── js/app.js          # كل منطق الواجهة
├── backups/               # النسخ الاحتياطية
├── backup.ps1 / backup.bat
├── scripts/               # سكريبتات مساعدة
└── README.md              # هذا الملف
```

## بدء التشغيل

```bash
cd blood-bank-server
node server.js
```

السيرفر يشتغل على port 3001.

## الوحدات الأساسية

### المخزون اليومي
- إدخال أرصدة الفصائل (A+, A-, B+, B-, O+, O-, AB+, AB-)
- عرض الجرد اليومي
- تقارير الاستهلاك الشهري
- تقارير الفرع

### العاملين
- إدارة بيانات العاملين (الاسم، الفئة، التصنيف، الوردية، التليفون، الرقم القومي)
- مراجعة شهرية (review)
- تصدير Excel/PDF

### شيت الجاهزية
- إدارة المناسبات (إضافة/تعديل/حذف)
- إدخال بيانات الجاهزية لكل مستشفى:
  - القوى العاملة (الاسم، التليفون، الورديات لكل يوم)
  - حالة الرصيد (كافي / غير كافي) مع عرض أرصدة الفصائل
  - مراجعة الصيانة (تتم / لا تتم)
  - الأعطال (يوجد / لا يوجد)
  - المستهلكات (كافية / غير كافية)
- جدول ملخص بجميع المستشفيات
- طباعة وتصدير PDF/Excel

### إدارة الأجهزة
- 22 نوع جهاز (فريزر بلازما، ثلاجة دم، ...)
- إدارة العدد والحالة والماركة والسعة لكل جهاز
- تصدير Excel

### المزامنة (للنسخة المحمولة)
- تصدير/استيراد قاعدة البيانات
- عرض حالة المزامنة

## النسخ الاحتياطي
